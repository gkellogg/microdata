Microdata Test Vocabulary

This is a vocabulary document used to define classes and properties used in Microdata Test Cases and associated test manifests.

This vocabulary extends the RDF Test Vocabulary and Test Manifest Vocabulary with Microdata-specific tests. The URI of the vocabulary is https://w3c.github.io/microdata/tests/vocab# (abbreviated by mdt in this document). Turtle and JSON-LD versions of the vocabular are also available. The vocabulary is published by the W3C Web Platform Working Group.

Vocabulary Terms

The vocabulary terms below constitute the complete Microdata Test vocabulary

Classes

mdt:NegativeEvaluationTest
Negative Evaluation Test

A Negative Evaluation test is successful when the result of processing the input file specified as mf:action results in an error or exception. The specifics of invoking test, including the interpretation of options (and other input files are specified through sub-class.
subClassOf: mdt:Test
mdt:NegativeJsonTest
Negative JSON Test

An error or exception should be generated when trying to generate JSON.
subClassOf:
  • rdft:TestEval
  • mdt:NegativeEvaluationTest
mdt:NegativeRdfaTest
Negative RDFa Test

An error or exception should be generated when trying to generate RDFa.
subClassOf:
  • rdft:TestEval
  • mdt:NegativeEvaluationTest
mdt:PositiveEvaluationTest
Positive Evaluation Test

A Positive Evaluation test is successful when the result of processing the input file specified as mf:action exactly matches the output file specified as mf:result using the comparison defined in another class. The specifics of invoking test, including the interpretation of options and other input files are specified through a sub-class.
subClassOf: mdt:Test
mdt:Test
Superclass of all Microdata tests

All Microdata tests have an input file referenced using mf:action. Positive and Negative Evaluation Tests also have a result file referenced using mf:result . Other tests may take different inputs and options as defined for each test class.
subClassOf:
  • rdft:Test
  • mf:ManifestEntry
mdt:ToJsonTest
To JSON Test

A ToJsonTest takes an input file (mf:accept) in text/html format to produce a an JSON file. The expected results are compared using JSON object comparison with the processor output.
subClassOf: mdt:PositiveEvaluationTest
mdt:ToRdfaTest
To RDFa Test

A ToRdfaTest takes an input file (mf:accept) in text/html format to produce a an HTML+RDFa file. The expected results are compared using DOM comparison with the processor output.
subClassOf:
  • rdft:TestEval
  • mdt:PositiveEvaluationTest

Properties

mdt:base
base

The base IRI to use when expanding or compacting the document. If set, this overrides the input document's IRI.
